# NOTES

Our final model is a 6-layer GPT (n_embd=160, n_head=8, block_size=256) totaling 1,929,280 parameters. We replaced the byte tokenizer with a 512-vocab BPE tokenizer to achieve 2.85x compression on Hindi text, which gives the model significantly more effective context per block. We also employed weight tying between the token embeddings and output projection, which saved 81,920 parameters that we reinvested into two additional transformer layers for greater depth. For the optimizer, we used AdamW with decoupled weight decay (0.1) on matrices, alongside a cosine learning rate schedule and linear warmup to stabilize early updates. To prevent gradient explosion in the deeper architecture, we added gradient clipping (max_norm=1.0) and a GPT-2-style scaled residual initialization. Finally, we reduced the batch size to 4 to eliminate Out-Of-Memory errors on the CPU during GELU forward passes. This configuration maximizes structural capacity and optimization efficiency within the strict 2,000-step and 2M-parameter constraints, resulting in a Dev BPB of 2.5219.
